# Codex Feishu Bridge 重构思路

## 目标
把当前桥接项目彻底简化，按新的交互逻辑重写，不复用旧项目命令体系。

## 新命令逻辑
### 1. 帮助命令
- `/help`
- `/帮助`

显示全部会话命令。

### 2. 查看目标列表
- `/list`
- `/列表`

返回当前所有可用目标。
目标的来源可以先用本地 Codex 线程列表生成，展示：
- 数字编号，例如 `101`这个“数槽”是一个默认绑定 workspace 和线程的一个统一的路由。

这样的话，就不需要再绑定某一个特定的 workspace 了，相当于做了统一绑定。
- 标题
- 工作区名
- 可选别名

### 3. 临时发消息到某个目标
- `#101 你的问题`
- `#小高 你的问题`

特点：
- 这条消息只发给指定目标
- 不改变默认目标
- 适合临时问某个线程

### 4. 切换默认目标
- `/go 101`
- `/切换 101`

把某个目标设为当前默认目标。
后续不带目标前缀的普通消息，都发给这个默认目标。

### 5. 查看当前默认目标
- `/where`
- `/当前`

显示：
- 当前默认目标编号
- 标题
- 工作区
- 路径
- 别名

### 6. 给当前目标起别名
- `/rename 小高`
- `/改名 小高`

给当前默认目标设置一个可记忆别名。
之后可直接：
- `#小高 你的问题`

### 7. 新建会话
- `/new [path]`

行为建议：
- 不带 path：在当前工作目录或默认目录新建会话
- 带 path：在指定目录新建会话
- 成功后自动切换为默认目标

## 普通消息逻辑
### 未设置默认目标时
普通消息返回提示：
- 先 `/list`
- 再 `/go 目标编号`
- 或直接用 `#编号 消息`

### 已设置默认目标时
普通消息直接发给当前默认目标。

## 必保留的核心逻辑
只保留下面四部分思路，不保留旧命令代码：

### A. 本地目标发现
从本地 Codex 数据中读取：
- workspace
- thread
- 标题
- 更新时间

### B. 会话映射
维护飞书聊天和默认目标之间的映射：
- chatId -> currentTargetId
- alias -> targetId

### C. 命令分发器
统一解析三类输入：
1. `/` 开头命令
2. `#目标 消息` 的临时路由
3. 普通消息

### D. Feishu 消息解析
需要兼容：
- text
- post
- @mention

## 第一版建议文件结构
仅供重写时参考：

- `src/main.ts`：入口
- `src/feishu.ts`：飞书接入
- `src/router.ts`：命令和消息分发
- `src/targets.ts`：目标发现与编号生成
- `src/session-store.ts`：默认目标与别名存储
- `src/codex.ts`：真正调用 Codex
- `src/types.ts`：公共类型
